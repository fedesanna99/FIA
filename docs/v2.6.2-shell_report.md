## 3. Decisione architettonica T9

Brief T9 chiede di "sostituire Layout con Shell". Implementato come
**condizionale**:

```ts
const useNewShell = activeId !== null && !isMobile && !isFocusMode;
```

- **Nuova Shell**: utente loggato + modello attivo + desktop ≥768px + NOT focus mode
- **Chrome legacy**: Dashboard (no model), mobile, focus mode

**Motivazione**:
- App.tsx ha 763 righe di logica intricata (listeners, dialog state, mobile UX, focus mode, drop zone, solver overlay) che NON sono in scope della Shell
- La Shell è il "telaio del workspace operativo" — quando l'utente sta lavorando su un modello
- Dashboard, Mobile e Focus mode hanno UX completamente diversi: convergeranno alla nuova Shell in Fase 3-5 in modo atomico

Trade-off:
- ✅ Zero rischio di regressione sui flussi non-model (Dashboard, Mobile, Focus)
- ✅ Tutti i dialog/wizard sempre disponibili (montati al root, fuori dal condizionale)
- ✅ Visual change visibile immediato quando l'utente lavora su un modello
- ⚠️ Coesistenza temporanea 2 shell (legacy + nuova) — è un costo accettabile per la Fase 2
