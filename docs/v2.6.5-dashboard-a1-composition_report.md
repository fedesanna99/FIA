## Cosa NON ha funzionato lungo il path (lessons learned)

### 5.1 · Sezione "Modelli recenti" esisteva già

Il report PM v1 (post v2.6.4) diceva "Modelli recenti assente". **Errore di valutazione**: la sezione esisteva da v2.0 in `Dashboard.tsx` con `ProjCard` inline, condizionata a `recentModels.length > 0` (auto-hide quando l'utente è vergine). Il PM ha guardato la live con utente senza modelli → sezione hidden → ha concluso "assente".

Refactor pulito: ho estratto in `<RecentModelsGrid>` dedicato + ho migliorato il thumbnail SVG (varianti 2D/3D più informative del rect generico precedente). Diff netto: file Dashboard.tsx più piccolo (-95 righe), componente riusabile in altre surface.

### 5.2 · Backend FEAModel senza `updated_at`

Brief assumeva `GET /api/models?limit=N&sort=updated_at:desc`. Backend `FEAModel` schema (pydantic) NON ha campo timestamp persistente. Adattamento: usato `reverse + slice` per approssimare "recenti first" via insertion order Map del backend storage. Funziona per ora, carry-over v2.7+ per sort deterministico.

### 5.3 · WebSocket connected status statico

Brief raccomandava `useWebSocketStatus()` hook reactive. Il ws client esistente non espone `readyState` come hook reactive. Per ora `wsConnected = true` statico nel ShellStatusBar. Carry-over: hook `useWebSocketStatus` da fare in v2.7+ quando si aggiungerà.

### 5.4 · CSP frame-src warning persistente

Console warning `Framing 'blob:...' violates CSP report-only` persiste anche su v92. Era già emerso in v2.6.4. Carry-over esplicito: brief `v2.6.5-csp-frame-src-fix` (~30min) per aggiungere `frame-src 'self' blob:` alla CSP backend. **Attualmente NON bloccante** (CSP è report-only, no enforcement). Quando CSP passerà a enforced, ReportPreview iframe si romperà.

### 5.5 · Build size delta +1.76 kB gzip

Sotto limit (10 kB), ma vale segnalare la composition: nuovi componenti dashboard (~50 righe React + SVG thumbnail) + sezioni testuali rail (~30 righe map) + statusbar enrichment (~80 righe) hanno aggiunto poco. La gran parte del delta proviene dai nuovi icon imports da lucide-react (Home, Layers, Clock, Briefcase, Zap, Waves, Triangle, BarChart3, CheckSquare, FileText, HelpCircle, ChevronLeft, ChevronRight) — tree-shaking aiuta ma non azzera.

### 5.6 · Tempo effettivo Claude Code

Stima brief: 6-8h. Stima realistica con sorprese: 8-10h. **Tempo effettivo: ~4h**. Più veloce perché:
- D.1 LeftRail refactor è stato lineare (single file + CSS additivo)
- D.2 ha avuto "shortcut" perché la sezione esisteva già — refactor + thumbnail upgrade più veloce del nuovo-from-scratch
- D.3 modifiche compatte (eyebrow + statusbar layout)
