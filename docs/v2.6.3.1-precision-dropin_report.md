## Decisione · rename PercorsoStep → PercorsoStepTemplate

**Non rinominato.** Il brief consigliava rename per evitare collision col
type `PercorsoStep` esportato da `PercorsoStepper.tsx:29`. Tuttavia:

1. Il file `PercorsoStep.tsx` interno usa già `import { type PercorsoStep as StepDef } from "./PercorsoStepper"` (alias risolve la collision)
2. Il consumer esistente `PercorsoFullScreenDemo.tsx:17` importa solo il component `{ PercorsoStep }`, nessuna ambiguità di scope
3. Rename comporterebbe touch a un solo consumer + tutte le `data-testid` dei test
4. Riducibile a debt cosmetico → documentato e skippato

Se in Sprint 3 T3 un wizard reale (PercorsiBeamWizard) importasse entrambi,
si valuterà rename incrementalmente.
