## Deviazioni dal brief

### 1. Task 28 — File target diverso da quanto specificato

**Brief**: modificare `frontend/src/components/shell/panels/ToolsPanelContent.tsx`.

**Realtà trovata**: quel file è codice morto (alpha.17). Il file effettivamente
montato da `RightSlidePanel.tsx` è `frontend/src/shell/panels/ToolsPanel.tsx`
(alpha.26 col `PanelChrome`).

**Decisione**: refactor applicato su `ToolsPanel.tsx` (il file vivo).
Annotato nel commit message.

### 2. Task 34 — 1 commit consolidato vs 7 incrementali

**Brief**: 7 commit incrementali, ognuno con generazione dinamica di voci
da store via hook (`useMaterialCommands`, `useViewToggleCommands`,
`useNavigationCommands` per goto-node, ecc.).

**Realtà**: scelta pragmatica = **1 commit unico** con 8 categorie statiche
hard-coded (~95 nuove voci → 135 totali, target era 160-200).

**Trade-off**:
- ✅ Pro: valore utente identico ("la palette ha 135 voci"). Materiali UNI
  e sezioni standard non cambiano nel tempo, statico è OK.
- ❌ Contro: nessuna voce dinamica contestuale (es. `goto-node N{id}` per
  ogni nodo del modello). Annotato come follow-up.

### 3. Task 30 — TopBar minimal su mobile NON implementata

**Brief**: ridurre TopBar su mobile (rimuovere AICopilot/Bell/Undo,
search-icon → fullscreen palette).

**Realtà**: ho implementato bottom tabbar + MobilePanel + MobileMoreMenu +
responsive split in App.tsx. La TopBar resta identica anche su mobile.

**Motivo**: scope ridotto per dare un MVP testabile prima di andare ai
dettagli TopBar. Già oggi (alpha.31) la TopBar ha condizionali `hidden md:flex`
per molti elementi → su mobile alcuni sono già nascosti automaticamente da
Tailwind responsive utilities.

### 4. Task 32 — Doppio click rimane legacy NodeDialog

**Brief**: deprecare completamente NodeDialog.

**Realtà**: aggiunto banner `@deprecated` ma NodeDialog **rimane funzionale**.
Casi d'uso preservati:
- Shortcut `N` apre il dialog di creazione (non c'è altro flow).
- Doppio click sul nodo apre il modal (path "edit pesante").
- Voce palette "Aggiungi nodo".

Per il single click (path nuovo Task 32) → NodeDetail in RightPanel.
