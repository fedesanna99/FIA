# Cluster B · audit caller toast/notify errori

**Data**: 2026-05-26
**Scope**: T1 brief v2.5.5 — mappare TUTTI i punti in cui il frontend mostra
un errore all'utente, prima di applicare i fix BUG-011/012/013 in T3.

## 1. Caller funzionanti (✓ già tradotti)

| Caller | File:riga | Pattern |
|--------|-----------|---------|
| **Interceptor axios globale** | `frontend/src/api/client.ts:60-67` | `if (status >= 400 && shouldToastHttpError(...)) → translateAxiosError + toast` |
| `useRunAnalysis` (analisi run) | `frontend/src/hooks/useAnalysis.ts:55-63` | catch manuale + `translateApiError(e?.response?.data ?? e)` + `notify("error", ...)` |

> **Scoperta architetturale critica**: l'interceptor `client.ts` mostra **già un toast tradotto** per ogni HTTP error 4xx/5xx, **prima** di rigettare la promise via `Promise.reject(err)`. Quando un `useMutation.onError: (e) => toast("error", "Errore X: ${e.message}")` viene chiamato, l'utente ha già visto il toast tradotto. Il toast generato dal `onError` è **duplicato e ridondante** — mostra "Request failed with status code 422" o simili invece del messaggio italiano.

## 2. Caller con bypass — assegnazione per scope T3

## 5. Finding aperti per sprint futuri (out of scope T3)

- **Ridondanza interceptor + onError**: 30 caller Categoria B mostrano toast duplicato. Cleanup ~1h in sprint debt.
- **`useLoadModel` non ha `onError`** (`hooks/useModel.ts:14-29`): se backend down, l'utente non vede toast esplicito di "load modello fallito". L'interceptor `client.ts` lo intercetta solo se non è in whitelist (`shouldToastHttpError`). Verificare se `["model", id]` rientra nelle eccezioni.
- **`NodeDialog`/`ElementDialog`/`LoadDialog`/`ConstraintDialog`** non hanno `onError` esplicito nelle mutation — affidano tutto all'interceptor. Verificare comportamento in caso di 5xx (server crash mid-mutation).
- **CommandPalette `apply-material/section`** propagano errori via `applyMaterialToSelection` callback. Da verificare il pattern di error reporting interno.

## 6. Strategia T3 finale

**Helper nuova**: `toastApiError(err, action)` in `lib/apiErrors.ts`. Forma:

```ts
export function toastApiError(err: unknown, action: string): void {
  const { title, description } = translateApiError(err);
  const full = description ? `${action}: ${title} · ${description}` : `${action}: ${title}`;
  toast("error", full);
}
```

Applicata ai 5 caller Categoria A. Pattern:
- BEFORE: `toast("error", \`Errore X: ${(e as Error).message}\`)`
- AFTER: `toastApiError(e, "Errore X")`

I 30 caller Categoria B + 7 caller Categoria C **non vengono toccati** in questo sprint.
