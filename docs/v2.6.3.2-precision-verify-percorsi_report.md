## T2 · Wire VerifyChecksLive in VerifyPanel · GIÀ FATTO

**File**: `frontend/src/shell/panels/VerifyPanel.tsx` (113 righe)

VerifyPanel ha hub-first navigation con 6 hub-cards (Live, EC2, EC3, EC5,
EC8, NTC18). Quando l'utente seleziona "Verifiche live" (`tab === "live"`)
viene renderizzato `<VerifyChecksLive />` che a sua volta compone:

```tsx
<div className="grid grid-cols-1 lg:grid-cols-[280px_1fr] gap-3 …">
  <ChecksRail checks={checks} activeId={activeId} onSelect={setActiveId} />
  <ChecksDetailTable
    checkId={activeSpec.id}
    title={activeSpec.detailTitle}
    subtitle={activeSpec.detailSub}
    rows={detailRows}
    ucLimit={1.0}
  />
</div>
```

Mapping dati:
- `useResultsStore.staticResults` → `sigmaMPa = max_stress / 1e6`
- 3 check normativi: S275 (`fyd=261.9`), EC3 (`fyd=235.0`), NTC18 (`fyd=261.9`)
- UC = `sigmaMPa / fyd` per ogni spec
- `state`: `pass` (UC<0.85), `warn` (UC<1.0), `fail` (UC>=1.0)
- Empty state con `FeatureButton featureId="run-static"` quando no results

Layout reattivo: 1 col mobile, 280px+1fr ≥lg.

## T3 · Wire PercorsoStep nei wizard percorso · DECISIONE

**Decisione**: `PercorsoStep` (template handoff) è già wirato in
`PercorsoFullScreenDemo.tsx` (159 righe, lazy-loaded in App.tsx via event
`feapro:open-percorso-fullscreen`). NON viene applicato a
`PercorsiBeamWizard.tsx` per le seguenti ragioni:

1. **PercorsiBeamWizard è dialog modale**, non pagina full-screen
   (è dentro `Dialog` con dismiss ESC/click-outside/swipe-back). Il
   template `PercorsoStep` è layout `h-full flex flex-col` con
   `<PercorsoStepper>` sticky → richiede full viewport non disponibile
   dentro Dialog.

2. **PercorsiBeamWizard usa già `PercorsoStepper` canonical** (riga 34
   `import { PercorsoStepper, PERCORSO_STEPS_6 }`). I 6 step sono
   allineati e lo Stepper è renderizzato in header del wizard.

3. **Refactor wizard → fullscreen route** non in scope (architectural
   change). Sprint successivo opzionale: convertire wizard in route
   `/percorso/beam/:step` e usare PercorsoStep template.

4. **PercorsoFullScreenDemo** è già la prova-funzionante del template
   handoff su 6 step C2-C7 (con help aside, validation chip, forward/back).

**Output utente**: il template handoff è dimostrato in app via
`window.dispatchEvent(new CustomEvent("feapro:open-percorso-fullscreen"))`.
PercorsiBeamWizard resta dialog by design.
