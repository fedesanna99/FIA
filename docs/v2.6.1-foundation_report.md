- Semantic: `--success #65A30D`, `--warn #B45309`, `--coral #C2410C`, `--purple #534AB7`, `--danger #DC2626`, `--info #0891B2`

- Shadow: `--shadow-pop`, `--shadow-elev`, `--shadow-card`, `--shadow-dialog`, `--shadow-hover`

**Dark** (`:root[data-theme="dark"]`):
- Surfaces dark: `--bg #0E1014`, `--bg-panel #16191F`, ecc.
- Ink dark: `--ink #F2F4F7`, ecc.
- Accent cyan brighter: `--accent #6EDDF5`
- Semantic + tinted backgrounds opacizzati per dark

**Universal** (`:root`):
- Radius: `--r-xs 4px` → `--r-3xl 20px`, `--r-full 9999px` (override 0 con `:root[data-radius="sharp"]`)
- Motion: `--d-fast 120ms`, `--d-mid 200ms`, `--d-slow 360ms`
- Fonts: `--font-sans` (Inter), `--font-display` (Plus Jakarta Sans), `--font-mono` (JetBrains Mono)
- Type scale: `--fs-xs 11px` → `--fs-5xl 56px`
- Letter spacing: `--ls-tight-1..4`, `--ls-wide-1..4`
- Spacing: `--sp-0..20` (4px scale)
- Z-index: `--z-viewport 0` → `--z-loader 70`

## 8. Provider useTheme — NOT wired

Hook `useTheme` creato e testato ma **NON applicato** in `App.tsx`. Decisione
del brief: il wiring del provider è scope Fase 2 (Shell). Motivazione: cambiare
`data-theme` ora invaliderebbe tutto lo stato visivo dei componenti precision
prima che il design new li sostituisca.

Per testare manualmente:
```js
document.documentElement.dataset.theme = "dark";  // o "light"
```

I componenti che usano `var(--bg)`, `var(--ink)`, ecc. dai nuovi token
reagiranno automaticamente.

```bash

```

dall'auto-mode classifier** anche se il brief lo autorizzava esplicitamente.

