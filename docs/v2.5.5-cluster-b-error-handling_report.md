## Comportamento BEFORE/AFTER

### BUG-046 + BUG-047 · Toaster position & max-width
- **BEFORE**: `fixed bottom-12 right-6 z-[200]` con `min-w-[300px] max-w-[360px]`. Toast in basso a destra, dimensione fissa, potenzialmente coperto da dialog modali con z-index alto (es. mappa, viewport overlay).
- **AFTER**: `fixed bottom-6 left-1/2 -translate-x-1/2 z-[9999]` con `w-[calc(100vw-3rem)] max-w-[480px]` + `pointer-events-none` sul container (con `pointer-events-auto` sui toast figli). Bottom-center, responsive width (480px desktop, full meno 48px margine su mobile), sopra qualsiasi UI tranne dialog modali al di sopra di z-9999.

### BUG-011 + BUG-012 · Errori JSON crudi
- **BEFORE** (5 caller): `toast("error", \`Errore X: ${(e as Error).message}\`)`. Su axios errors mostrava "Request failed with status code 422" o stack trace. Su errori jsPDF/SheetJS mostrava il messaggio nativo della libreria (spesso in inglese tecnico).
- **AFTER** (5 caller): `toastApiError(e, "Errore X")`. Estrae il payload `e.response?.data` quando disponibile e lo passa a `translateApiError` → italiano umano. Per errori non strutturati, fallback su `e.message` o stringa nuda. Nessun `[object Object]`, nessun raw axios error message.

### BUG-013 · Errori senza contesto operativo
- **BEFORE**: `"Aggiungi almeno un vincolo prima di lanciare l'analisi"` mostrato senza prefisso azione. L'utente non sa se l'errore è dell'analisi corrente o di altro.
- **AFTER**: `"Errore analisi: Aggiungi almeno un vincolo prima di lanciare l'analisi"`. Il prefisso azione (`action` arg di `toastApiError`) fornisce contesto operativo immediato.

### Handler globali (S0 T2 follow-up)
- **BEFORE** (S0 T2): `console.error("[global error]", ...)` + niente toast. L'utente non vedeva errori native catch-all (promise rejection, errori window, ResizeObserver loop, ecc.).
- **AFTER**: stesso `console.error` + `toastApiError(err, "Errore imprevisto")` differito via `queueMicrotask` (per attendere mount Toaster). Filtro per rumore noto: `ResizeObserver loop` e `Loading chunk N failed` vengono soppressi a `console.warn` senza toast (sono noti spurii browser, infastidirebbero l'utente).
