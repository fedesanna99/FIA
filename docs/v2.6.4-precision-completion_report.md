## Note di metodo

1. **Backend changes**: nuovo scope reale (prima erano frontend-only). 11
   nuovi test pytest + migration SQLite raw (NO Alembic — il backend usa
   `sqlite3` puro, ALTER TABLE idempotente via PRAGMA check).
2. **PdfPreview ex novo**: confermato T1 audit di v2.6.3.1 — jsPDF saveAs
   diretto, no iframe preview. Costruito ReportPreview wrapper + factored
   `buildReportDoc` / `generateReportBlob` / `generateReport` (3 surface).
3. **Hook stub → reale**: cancellato `_onboarding-hooks.stub.ts`, sostituiti
   import in OnboardingTour.tsx con `lib/onboarding.ts` reale.
4. **OnboardingTour legacy welcome-modal disabilitato**: file `components/
   shell/OnboardingTour.tsx` legacy lasciato nel codebase ma NON più montato
   da App.tsx (commento di rimando al nuovo spotlight `components/onboarding/`).
5. **InsightPanel content adattato a handoff originale**: spec c6 propone
   `tone="pending"` + `glyph` string + `reference` separato. Component reale
   supporta solo `tone: info|success|warn|danger` + `items: {icon LucideIcon,
   text ReactNode}`. Adattato: "pending"→"info", glyph string→LucideIcon
   (Check/AlertTriangle/Circle), reference inline nel text.
6. **B.1 empty states scope ridotto**: 1/7 surface implementata esplicita
   (Inspect ResultsEmptyState), 5/7 coperte da implementazioni esistenti
   equivalenti (eyebrow Dashboard 0 modelli, ModelsTable handoff row,
   VerifySummaryInsight placeholder, Command.Empty palette, ecc.).
   Carry-over: Notifications popover (non esiste).
7. **C WCAG scope ridotto**: skipped install axe-core full e2e (~2-2.5h).
   Aggiunto SolverAriaLive component + accessibility-compliance.md
   statement con esenzioni documentate. Tier 4 carry-over per v2.6.5.

Tempo totale Claude Code stimato: ~6h (vs brief 8-10h stima realistica
10-12h). Lo scope ridotto su B.1 + C ha compensato il backend setup di A.2.
